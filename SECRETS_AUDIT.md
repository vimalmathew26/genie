# Secrets Audit — genie-public

Scope: every `.py` file and `requirements.txt` in `genie-public` (37 files, the Step-1
approved manifest). Scanned for API key patterns (`sk-or-`, `sk-`, `api_key`, `API_KEY`,
`token`, `TOKEN`, `secret`, `password`, `bearer`), literal key formats
(`sk-or-v1-...`, `ghp_...`, Telegram `digits:token`), hardcoded absolute paths,
usernames, emails, and personal URLs. Nothing was modified — this is a report only.

## Result: no live secrets found

The codebase already follows the correct pattern everywhere a secret is used: read
from the environment, never hardcoded.

| Secret | Where read | Pattern |
|---|---|---|
| OpenRouter API key | [config.py:28](config.py#L28) `OPENROUTER_API_KEY = os.getenv("OPENROUTER_API_KEY", "")` | env var, empty default |
| OpenRouter API key (direct callers) | [llm_client.py:120](llm_client.py#L120), [scratchpad_writer.py:325](scratchpad_writer.py#L325), [vision_mixin.py:317,460,613](vision_mixin.py#L317) | all import `OPENROUTER_API_KEY` from `config`, never re-declare it |
| GitHub token | [github_actions.py:43,106](github_actions.py#L43) `token = os.environ.get("GITHUB_TOKEN", "")` | env var, empty default, raises `RuntimeError` if missing |
| GitHub token (test suite) | [genie_phase5.py:60](genie_phase5.py#L60) `_T36_GITHUB_TOKEN = os.environ.get("GITHUB_TOKEN", "")` | env var |
| Telegram bot token / chat ID | [config.py:759-760](config.py#L759) `os.getenv("TELEGRAM_BOT_TOKEN", "")` / `os.getenv("TELEGRAM_CHAT_ID", "")` | env var |
| Telegram bot token (review_bot.py) | [review_bot.py:30](review_bot.py#L30) imports from `config` | no re-declaration |

No literal key values matching `sk-or-v1-…`, `ghp_…`, `gho_…`, or Telegram's
`\d{9,10}:\w{35}` bot-token shape were found anywhere in the tree.

## What Step 3 still needs to do (process hygiene, not fixes for live leaks)

Even though nothing here is currently leaking, a couple of things are prerequisites
for the repo to actually run once cloned by someone else, and for defense-in-depth:

1. **`.env.example`** doesn't exist yet — needs to be created listing every env var
   the code reads: `OPENROUTER_API_KEY`, `GITHUB_TOKEN`, `TELEGRAM_BOT_TOKEN`,
   `TELEGRAM_CHAT_ID`, `GENIE_VERIFY` (optional, defaults to `"1"`), and the test-only
   `GITHUB_TEST_REPO` used by `genie_phase5.py`.
2. **`.gitignore`** doesn't exist yet — needs `.env`, `__pycache__/`, `*.pyc`, `logs/`,
   `venv/`/`.venv/`, and IDE cruft, so a future local run never accidentally commits
   secrets or logs.
3. No absolute-path or personal-info replacements are actually required — the audit
   found none — but Step 3's re-scan will re-confirm zero hits after `.gitignore`/
   `.env.example` are added.

## Summary

**0 live secrets. 0 hardcoded personal paths/emails/usernames.** The codebase was
already written env-var-first. Step 3 is essentially just adding `.env.example` and
`.gitignore` as safety nets, then re-running this same scan to confirm zero hits.
